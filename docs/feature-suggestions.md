# Feature Suggestions For ESTT Community

This document proposes new features for the current ESTT Community website.
The goal is to build on what already exists today: resources, clubs, events,
chat, notifications, ads, profiles, admin moderation, and the mobile/PWA experience.

## Product Direction

The website is already more than a simple resource-sharing platform.
The next wave of features should push it further in three directions:

1. Make academic use more personalized and useful every day.
2. Make campus life flows smoother for clubs, events, and communication.
3. Give admins better moderation, analytics, and automation tools.

## Priority 1: High Impact Features

### 1. Smart resource recommendations

Show recommended resources based on:

- filiere
- semester / academic level
- recently viewed modules
- top-rated resources in the same module

Why this matters:

- helps new students discover useful content faster
- increases engagement on existing resources
- makes the platform feel personalized

Suggested additions:

- "Recommended for you" block on home and profile
- "Popular this week in your filiere"
- "Continue where you left off"

### 2. Favorites, bookmarks, and study collections

Allow users to:

- bookmark resources
- organize them into folders or collections
- share a collection with classmates

Examples:

- "S2 revision pack"
- "Best GI networking resources"
- "Exam prep for database"

Why this matters:

- gives students a reason to come back regularly
- makes saved content more useful than a flat favorites list
- supports collaborative studying

### 3. Better event participation flow

Expand the event system with:

- RSVP states: interested, going, attended
- reminder notifications before events
- waitlist support when capacity is reached
- post-event recap with photos, slides, and feedback

Why this matters:

- improves turnout for clubs
- reduces confusion before events
- turns events into reusable community content

### 4. Stronger chat experience

The chat already exists, so the next upgrades should be:

- pinned messages
- message reactions
- mentions
- better thread handling
- "answered" or "useful" markers in help discussions

Why this matters:

- turns chat from simple conversation into a real help space
- helps mentors and active students stand out
- makes useful answers easier to find later

### 5. Full user dashboard

Create a richer personal dashboard with:

- recent activity
- upcoming events
- saved resources
- unread notifications
- pending contributions
- clubs joined

Why this matters:

- gives each student a personal home inside the platform
- reduces the need to jump between many pages
- increases retention

## Priority 2: Community And Campus Features

### 6. Mentor and peer-help system

Introduce a lightweight mentoring layer:

- verified mentors by filiere / year
- mentor badge in chat and profile
- "Ask a mentor" section
- office-hour style availability

Why this matters:

- supports first-year students
- rewards experienced contributors
- adds more structure to community help

### 7. Club recruitment and application workflows

For clubs, add:

- recruitment campaigns
- application forms with statuses
- applicant review dashboard
- interview / shortlist notes for club admins

Why this matters:

- matches the way clubs actually operate
- reduces off-platform coordination
- gives clubs a professional workflow

### 8. Club member roles and permissions

Add structured role management inside clubs:

- president
- vice president
- media manager
- event manager
- treasurer
- moderator

Why this matters:

- avoids all permissions being concentrated in one person
- makes club administration safer and clearer
- prepares the platform for larger clubs

### 9. Event attendance certificates and history

Build on the existing ticket/certificate flows with:

- attendance history in the user profile
- downloadable event certificates
- organizer validation logs
- club performance history

Why this matters:

- gives students a portfolio of engagement
- adds value for clubs hosting serious events
- helps justify participation and learning outside class

### 10. Resource discussion and Q&A per module

Add discussion spaces connected to modules or resources:

- ask questions on a specific module
- comment on a resource
- mark one answer as helpful
- show top contributors per module

Why this matters:

- adds academic context around static files
- encourages deeper participation
- helps surface the best explanations

## Priority 3: Marketplace And Monetization Features

### 11. Better student ads marketplace

Improve the ads area with:

- categories
- price filters
- location or campus relevance
- seller profiles
- ad reports
- trust indicators

Why this matters:

- makes the ads space more usable
- increases user trust
- improves monetization quality

### 12. In-platform conversations for ads

Let users contact ad publishers directly through the platform.

Suggested features:

- message seller
- quick safety prompts
- unread conversation count
- archive / block conversation

Why this matters:

- reduces friction
- keeps traffic inside the platform
- gives moderation a better trail when problems happen

### 13. Sponsored placements and analytics for advertisers

For the ads portal, add:

- performance stats
- clicks
- impressions
- CTR
- campaign duration history
- boosted placement option

Why this matters:

- makes the ad product more professional
- helps justify ad pricing
- gives advertisers feedback instead of blind spending

## Priority 4: Admin, Moderation, And Trust

### 14. Unified moderation inbox

Give admins one place for:

- pending resources
- reported ads
- bug reports
- club requests
- flagged users
- pending edits

Why this matters:

- reduces admin overhead
- prevents requests from being missed
- makes operations easier as the platform grows

### 15. Audit logs for sensitive actions

Track admin actions such as:

- approval / rejection
- content deletion
- role changes
- settings changes
- ad activation

Why this matters:

- improves accountability
- helps debug mistakes
- is useful if multiple admins work together

### 16. Better safety and reporting tools

Add reporting options for:

- spam
- inappropriate content
- misleading ads
- harassment in chat
- fake resources

Why this matters:

- makes the platform safer
- gives users confidence
- helps moderation scale

## Priority 5: Mobile, PWA, And Experience

### 17. Offline-friendly academic mode

Make the PWA more useful offline by caching:

- recently visited resources
- event summaries
- favorite items
- essential profile data

Why this matters:

- students often have inconsistent connectivity
- makes the app feel more native
- improves day-to-day reliability

### 18. Guided onboarding for first-time users

Add onboarding based on the current product:

- choose filiere
- choose year / semester
- follow clubs
- save first resources
- join the right chat

Why this matters:

- reduces confusion
- helps users discover more features
- improves activation for new students

### 19. Better notification preferences

Allow users to control notifications by type:

- contributions
- events
- club announcements
- chat replies
- ads messages
- admin alerts

Why this matters:

- prevents notification fatigue
- makes alerts feel relevant
- improves trust in the notification system

## Nice-To-Have Ideas

### 20. Semester planner

Let users build a light academic planner with:

- modules
- deadlines
- exam reminders
- study checklists

### 21. Polls for clubs and community

Useful for:

- choosing event dates
- collecting opinions
- running quick votes inside clubs

### 22. Public contributor leaderboard

Display helpful rankings such as:

- top contributors this month
- top mentors
- most useful resources

This should be optional and balanced so it motivates without becoming noisy.

### 23. Short-form campus feed

Create a lightweight feed for:

- quick announcements
- mini updates from clubs
- important campus reminders
- featured student work

### 24. Resource quality score

Generate a simple quality score using:

- ratings
- clicks
- saves
- reports
- approval history

This can help ranking and moderation.

## Suggested Roadmap

### Phase 1

- smart recommendations
- favorites and collections
- better event participation flow
- richer user dashboard

### Phase 2

- improved chat
- mentor system
- club recruitment workflows
- notification preferences

### Phase 3

- ads conversations
- ad analytics
- unified moderation inbox
- audit logs

### Phase 4

- offline mode
- planner
- module Q&A
- contributor reputation features

## Final Recommendation

If only a few features should be built first, the best bets are:

1. Favorites and study collections
2. Smart resource recommendations
3. Better event RSVP and reminders
4. Richer user dashboard
5. Improved chat with mentions, reactions, and stronger threads

These fit the current product naturally, improve retention, and make the platform feel much more complete without requiring a full product reset.
